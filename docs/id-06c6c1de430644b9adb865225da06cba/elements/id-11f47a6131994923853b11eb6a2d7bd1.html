Het (near) real-time ophalen van gegevens vindt plaats via RESTful API’s ()
RESTful API’s zijn inmiddels de breed geaccepteerde wijze voor het ontsluiten van gegevens. Ze maken het eenvoudig om gegevens (near) real-time te delen mechanisme om, gebaseerd op web-standaarden. RESTful API's bieden geen standaard mechanismen voor het gegarandeerd afleveren van gegevens en passen niet goed op een gebeurtenisgedreven architectuurstijl.
ImplicatiesAanbieders realiseren gegevensdiensten voor het (near) real-time ophalen van gegevens als RESTful API’s, tenzij hele specifieke eisen worden gesteld vanuit de context of het gebruik van deze gegevens.
ImplicatiesAlternatieve protocollen voor het ophalen van gegevens zoals GraphQL kunnen naast RESTful API’s worden aangeboden.
ImplicatiesAPI’s worden kenbaar gemaakt in relevante catalogi zoals developer.overheid.nl, data.overheid.nl en/of het Nationaal Geo Register.
ImplicatiesAPI’s voldoen aan de Nederlandse API-strategie en de REST API Design Rules.
